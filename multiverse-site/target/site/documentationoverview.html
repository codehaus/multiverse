Recommended Papers

A lot of research has been done on transactional memory by a lot of very smart people. Multiverse relies on a lot of concepts and techniques discovered and invented by these people. This is a listing of the most influencal papers for Multiverse.
  1. Transactional Locking II:contains a lot of information about the general principles behind the AlphaStm implementation (the main STM implementation of Multiverse)

Recommended Books

  1. Transactional Information Systems
  2. Transactional Memory
  3. The Art of Multiprocessor Programming not only contains a lot of information about the theories behind concurrent programming, but also has a chapter about Software Transactional Memory
  4. Concurrent Programming in Java
  5. Java Concurrency in Practice 'the' book about practical concurrent programming in Java.
  6. Expert Oracle Database Architecture: 9i and 10g Programming or Expert One on One - Oracle written by Thomas Kyte. These books contain a lot of information about low level techniques used in databases but also provide a lot of information about a very scalable concurrency approach called: Multi Version Concurrency Control (MVCC) that is also used in MySQL (with InnoDB) and Postgresql. Multiverse applies the general principles of MVCC (hence the name Multiverse).
