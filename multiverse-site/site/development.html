Downloading the sources

If you are not a committer you can execute the following command: 
git clone git://git.codehaus.org/multiverse.git multiverse
This command can be executed in the directory you want the Multiverse project to be checked out. The multiverse directory automatically is created for you.

If you are a committer you can execute the following command: 

git clone ssh://git@git.codehaus.org/multiverse.git
If you execute this command, and you haven't registered a 'Authorized Keys' at Codehaus, you will be asked for a password. I don't know what the password is and it is best to create an key and upload it to codehaus. For more information about creating a key, check the following SSH Guide on Codehaus and the public part of the key can be copy/paste to your Codehaus details page A better manual will be provided in the future because this 'getting up and running' stuff always irritates me and wastes my precious time.

Building the project

You need to install Gradle 0.9 Preview 1 (download). If you go to the root directory and type 'gradle -i' the project will automatically be compiled and the artifacts will also be stored in your local maven repository. More important commands: 
    //builds the jar from scratch (unit tests are not executed)
    gradle -i clean install

    //runs the tests
    gradle -i test

    //runs the integration tests (this can some time).
    gradle -i integrationtest
The integrationtests take a serious amount of time (more than an hour). In the 0.6 release it is going to be possible to configure from the buildscript/commandline how long the tests should run. Also make sure that you have Java 1.6 or higher installed; Sun/IBM/Open(6,7)-JDK should work without a problem since they are tested.

Setting up IDEA

Currently the Maven POMS are still available and kept in sync with the Gradle script, so you can use that as a bases when creating a Idea project.

Setting up Eclipse

Currently the Maven POMS are still available and kept in sync with the Gradle script, so you can use mvn eclipse:eclipse to generate the required Eclipse project files.
