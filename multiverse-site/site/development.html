Downloading the sources

If you are not a committer you can execute the following command: 
git clone git://git.codehaus.org/multiverse.git multiverse
This command can be executed in the directory you want the Multiverse project to be checked out. The multiverse directory automatically is created for you.

If you are a committer you can execute the following command: 

git clone ssh://git@git.codehaus.org/multiverse.git
If you execute this command, and you haven't registered a 'Authorized Keys' at Codehaus, you will be asked for a password. I don't know what the password is and it is best to create an key and upload it to codehaus. For more information about creating a key, check the following SSH Guide on Codehaus and the public part of the key can be copy/paste to your Codehaus details page A better manual will be provided in the future because this 'getting up and running' stuff always irritates me and wastes my precious time.

Building the project

If you have installed Maven 2, the only thing you need to do is the execute 'mvn clean install' in the root Multiverse directory. Make sure that you have Java 1.6 or higher installed.

Running the stress tests

There are a bunch of stress tests (they are named *StressTest) that run for quite some time (15/20 minutes on my laptop). They are not executed under a standard phase cause of Clover (the additional instrumentation caused by clover takes a lot of extra time). At the moment the stress tests are completely disabled, so you need to run them from your ide.

Setting up Clover

Multiverse uses Clover as test coverage tool and out of the box a evaluation license is provided. This license eventually will expire, so you will need a real license. Luckily Atlassian was kind enough to provide a free license for the Multiverse project. Please send me (alarmnummer at gmail dot com) and email if you are a committer, and I'll send you the key.
