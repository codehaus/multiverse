Getting Started

This document contains some hints on how to quickly get started with a Multiverse project.
  1. Use the Maven archtype: From the command line, run
    mvn -DarchetypeGroupId=org.multiverse -DarchetypeArtifactId=multiverse-project-archetype -DarchetypeVersion=multiverse-version archetype:generate
    to create a Maven project setup for Multiverse development. Just import into your favourite IDE and have a look at the included sample code!
    If you are offline and thus aren't able to access the repositories hosting the archetype, you can install the provided JAR locally by executing
    mvn install:install-file -DgroupId=org.multiverse -DartifactId=multiverse-project-archetype -Dversion=multiverse-version -Dpackaging=jar -Dfile=archetype/multiverse-project-archetype-multiverse-version.jar
    before continuing as above. You can also add the multiverse-alpha-multiverse-version-jar-with-dependencies.jar JAR required by the project to your repository in this way, if you are offline.
  2. Remember that tests require the Multiverse agent: The archetype's Surefire plugin configuration already takes care of starting the Multiverse agent before running your tests from Maven. If you intend to run them "by hand ", for instance in Eclipse, you will need to add the required VM arguments to start the agent.
    In Eclipse, this means opening the Run Configuration for the test you're trying to run, and adding
    M2_REPO/org/multiverse/multiverse-alpha/multiverse-version/multiverse-alpha-multiverse-version-jar-with-dependencies.jar
    to the VM arguments tab.
